# üöÄ Deployment Guide - Gohar Dental Clinic Website

This guide provides step-by-step instructions to deploy your website to Vercel.

## üìã Pre-Deployment Checklist

Before deploying, ensure you have:
- ‚úÖ All code committed to a Git repository (GitHub, GitLab, or Bitbucket)
- ‚úÖ Node.js 18.x or higher installed
- ‚úÖ Tested the website locally (`npm run dev`)
- ‚úÖ Built successfully (`npm run build`)
- ‚úÖ A Vercel account (free to create at [vercel.com](https://vercel.com))

## üéØ Method 1: Deploy via Vercel Dashboard (Recommended for Beginners)

### Step 1: Prepare Your Repository

1. **Push your code to GitHub:**
   ```bash
   cd gohar-dental-clinic
   git init
   git add .
   git commit -m "Initial commit: Gohar Dental Clinic website"
   git branch -M main
   git remote add origin YOUR_GITHUB_REPO_URL
   git push -u origin main
   ```

### Step 2: Deploy on Vercel

1. **Go to [vercel.com](https://vercel.com)** and sign in/sign up

2. **Click "Add New Project"**

3. **Import your repository:**
   - Select your Git provider (GitHub/GitLab/Bitbucket)
   - Find and select the `gohar-dental-clinic` repository
   - Click "Import"

4. **Configure your project:**
   - **Project Name**: `gohar-dental-clinic` (or your preferred name)
   - **Framework Preset**: Next.js (auto-detected)
   - **Root Directory**: `./` (leave as default)
   - **Build Command**: `npm run build` (auto-detected)
   - **Output Directory**: `.next` (auto-detected)
   - **Install Command**: `npm install` (auto-detected)
   - **Node Version**: 18.x or higher

5. **Environment Variables (Optional):**
   
   If you want to add analytics or other services, click "Environment Variables" and add:
   ```
   NEXT_PUBLIC_GA_ID=your-google-analytics-id
   ```

6. **Click "Deploy"**

7. **Wait for deployment** (usually takes 2-3 minutes)

8. **Your site is live!** You'll get a URL like: `https://gohar-dental-clinic.vercel.app`

### Step 3: Add Custom Domain (Optional)

1. **In your Vercel project dashboard**, go to "Settings" ‚Üí "Domains"

2. **Add your custom domain:**
   - Enter your domain (e.g., `gohardental.com`)
   - Click "Add"

3. **Configure DNS settings** with your domain provider:
   - Add the DNS records shown by Vercel
   - Wait for DNS propagation (5 minutes to 48 hours)

4. **SSL certificate** is automatically generated by Vercel

## ‚ö° Method 2: Deploy via Vercel CLI (For Advanced Users)

### Step 1: Install Vercel CLI

```bash

```

### Step 2: Login to Vercel

```bash

```

### Step 3: Deploy

```bash
cd gohar-dental-clinic

```

Follow the prompts:
- **Set up and deploy**: Yes
- **Which scope**: Select your account
- **Link to existing project**: No
- **Project name**: gohar-dental-clinic
- **Directory**: `./` (press Enter)
- **Override settings**: No (press Enter)

### Step 4: Production Deployment

For production deployment:
```bash
vercel --prod
```

## üîß Post-Deployment Configuration

### Update SEO Metadata

After deployment, update the `metadataBase` URL in `app/layout.tsx`:

```typescript
metadataBase: new URL('https://your-actual-domain.com'),
```

### Google Search Console

1. **Go to [Google Search Console](https://search.google.com/search-console)**
2. **Add your property** (your deployed URL)
3. **Verify ownership** using the meta tag method
4. **Submit your sitemap**: `https://your-domain.com/sitemap.xml`

### Update Contact Form (Optional)

To make the contact form functional:

1. **Choose an email service:**
   - [SendGrid](https://sendgrid.com/)
   - [Resend](https://resend.com/)
   - [EmailJS](https://www.emailjs.com/)

2. **Create an API route** in `app/api/contact/route.ts`:

```typescript
import { NextResponse } from 'next/server';

export async function POST(request: Request) {
  const data = await request.json();
  
  // Add your email service integration here
  // Example: await sendEmail(data);
  
  return NextResponse.json({ success: true });
}
```

3. **Update the form submission** in `components/sections/ContactSection.tsx` to use the API route

## üêõ Troubleshooting

### Build Fails

**Error: Module not found**
```bash
# Solution: Clear node_modules and reinstall
rm -rf node_modules package-lock.json
npm install
npm run build
```

**Error: TypeScript errors**
```bash
# Solution: Check for type errors
npm run lint
# Fix any reported errors
```

### 3D Models Not Loading

If 3D elements don't appear:
1. Check browser console for errors
2. Ensure Three.js is in dependencies (not devDependencies)
3. Verify `next.config.ts` has `transpilePackages: ['three']`

### Slow Performance

1. **Enable Vercel Analytics** to identify bottlenecks
2. **Optimize images**: Use Next.js Image component
3. **Check Lighthouse scores**: Run audit in Chrome DevTools

## üîê Security Best Practices

1. **Enable HTTPS** (automatic on Vercel)
2. **Set up security headers** in `next.config.ts`:

```typescript
async headers() {
  return [
    {
      source: '/(.*)',
      headers: [
        {
          key: 'X-Frame-Options',
          value: 'DENY',
        },
        {
          key: 'X-Content-Type-Options',
          value: 'nosniff',
        },
      ],
    },
  ];
}
```
